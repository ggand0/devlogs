### 2. GPU decode benchmarking — found it's slower than SW

Added `--bench <video>` CLI flag for raw decode throughput measurement.
Results at 640x480:

| Codec | SW fps | GPU fps |
|-------|--------|---------|
| AV1   | 312    | 162     |
| H.264 | 681    | 313     |

GPU is 2x slower due to the readback path (GPU decode → CPU copy → NV12→RGBA
conversion → GPU texture upload). The fix is vk-video's `WgpuTexturesDecoder`
(zero-copy, frames stay as wgpu textures) but that's a separate effort.

**Decision:** Don't merge GPU decode branch. SW decode is sufficient.
GPU decode infrastructure preserved for future zero-copy work.

### 5. Grid view (feat/grid-view, merged to main)

Core feature: play multiple episodes simultaneously in a tiled grid.

**Architecture:**
- `GridView` manages N `GridPane` instances, each with its own `VideoPlayer`
  (independent SW decode thread + bounded channel)
- `GridDataset` struct bundles dataset references to avoid 4+ args everywhere
- Shared `rebuild()` helper for navigate/jump/resize operations
- Manual rect math rendering via `ui.painter()` — no egui layout inside grid

**Features:**
- Toggle with G key or View menu
- Grid size picker (hover-to-preview mini grid, like Google Docs table insert)
- Supports non-square layouts (e.g. 3x2) and up to 10x10
- Episode list stays visible, highlights all grid episodes
- Auto-scroll to show full selected range via `scroll_to_rect`
- Drag-drop dataset reload while in grid mode
- Click pane to select (highlighted border)

**Performance (H.264 640x480):**
- 2x2: ~72 fps
- 3x3: ~60 fps
- 5x5: ~30 fps
- 6x6: ~22 fps
- 10x10: ~8-10 fps

**Bottlenecks identified (not fixed yet, documented in devlog 014):**
1. `ctx.load_texture()` creates new GPU texture per frame per pane — should
   reuse via `texture.set()`
2. `frame_to_color_image()` per-pixel Color32 loop — could be zero-cost
   reinterpret since Color32 is [u8;4] same as RGBA
3. swscale Context created per frame — should cache per decode thread

### 6. Renderer switch

Switched from eframe's default glow (OpenGL/GLX) to wgpu (Vulkan on Linux,
Metal on macOS, DX12 on Windows). The glow backend was failing with GLX
`BadValue` errors. Added explicit `wgpu`, `x11`, `wayland` features.

### 7. Cross-platform testing

- **macOS:** works after bumping ffmpeg-next 7→8
- **Windows:** works after installing LLVM/Clang + FFmpeg shared build

README updated with prerequisites for all three platforms.
